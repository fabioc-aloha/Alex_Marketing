## Multi-Document Consistency Framework

### Cross-Reference Validation Protocol

When maintaining multiple marketing documents:

1. **Feature Claims** - Same features listed identically across all docs
2. **Pricing** - Consistent tier names and prices (Free/$9 Pro/$29 Team/Custom Enterprise)
3. **Statistics** - Same numbers cited everywhere (update all or none)
4. **Voice** - Alex speaks in first person consistently
5. **Visual Style** - Same colors, fonts, diagram styles

### Consistency Checklist

```markdown
Before publishing any marketing content:
- [ ] Feature list matches across all channels
- [ ] Pricing unchanged or updated everywhere
- [ ] Screenshots current with actual UI
- [ ] Links tested and working
- [ ] Alex voice maintained (first person, confident, transparent)
- [ ] Brand colors correct (#007ACC, #4CAF50)
```

## Optimal Posting Schedule

### Best Times (PST)

| Platform | Day | Time | Reason |
|----------|-----|------|--------|
| Reddit | Tue-Thu | 9-11 AM | Developer morning routine |
| Twitter | Tue-Thu | 10 AM-1 PM | Peak tech engagement |
| Dev.to | Tue-Thu | 8 AM-12 PM | Article discovery window |
| LinkedIn | Tue-Wed | 9-10 AM | Professional browsing |
| YouTube | Sat-Sun | 10 AM | Weekend learning time |

### Avoid
- **Weekends** for text content (low dev engagement)
- **Friday afternoons** (people checking out)
- **Mondays** (inbox overload)
- **Major holidays** (attention elsewhere)

### Post Spacing
- Minimum 3 days between Reddit posts to same subreddit
- Maximum 2-3 tweets per day
- Weekly newsletter maximum
